    Introduction

    Welcome to the ModNet API documentation! We hope you build amazing applications with our Registry API.

    Most resources can also get a JSON response if you append &api to it

    Authentication

    To authorize, use this code:

    # Please make sure you have the MineModWrapper loaded as library

MineMod = new MineMod(`apikey`);


    # With shell, you can just pass the correct header with each request
curl "http://api.minemod.com/"
  -H "Authorization: apikey"

    Make sure to replace apikey with your API key. Note that our SSL certificate may change.

    MineMod makes use of API keys to secure our API’s and stop abuse MineMod expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: apikey

    You must replace `apikey` with your personal API key.

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files…just save them to the `includes` folder and add them to the top of your `index.md`’s frontmatter. Files are included in the order listed.

    The MineMod web API may return some errors

    Error Code Meaning
    400 Bad Request – We don’t even know what you are trying to do.
    401 Unauthorized – Your API key is wrong
    403 Forbidden – This resource is protected
    404 Not Found – The specified mod(pack) could not be found
    405 Method Not Allowed – You tried to do stuff in the wrong way.
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The modpack requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re sending us too many requests, slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.

    The Wrapper

    The MineMod Minecraft Wrapper is a very usefull utility we use to make sure our clients always have the latest and greatest versions installed. It works by reading some command line arguments it gets send from the Minecraft Launcher

    Argument What does it do
    -mod-packid Specify the modpack to download.
    -mod-versionid install version (id) instead of the latest version
    -mod-cleardb clear the local modindex
    -mod-nolog disable crash report uploading
    -mod-silent SSSSH, don’t log anything!
    -mod-noresolve Disable dependency resolving for mods.
    -mod-nodownload Disable the downloading of resources from our servers. (excluding API requests)
    -mod-offline Disable all contact with our servers.
    -mod-url Overwrite the API urls our launcher uses
    -mod-apikey Use a special API key for talking to the API
    -mod-norelaunch Install the pack. but don’t launch minecraft when done
    -mod-loglevel (1-5) enable log levels
